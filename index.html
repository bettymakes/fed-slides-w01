HTML

HyperText Markup Language

Markup Basics

HTML stands for Hypertext

Language used to describe your document on the World Wide Web

HTML History

  • Invented by Sir Tim Berners-Lee
  • Created the first web browser
  • Wanted to share academic papers in a digital format
- Tim Berners Lee is the 'god-father' of the world wide web, he invented it

- Came from academic background, wanted to share academic papers with others in this digital format

- So he has a document, with a specific formatting: paragraphs, headers, tabular data etc.

- Needed to find a way to describe these documents, purpose being to provide meaning to a computer in terms of what content its rendering.

- Came up with HTML. That's what the markup language was created for, to semantically describe your documents.

HTML Tag Anatomy

<p id="unique" class="group">Some Content</p>

This is the opening tag.

  • Starts with a tag name p in this case
  • HTML tags always start with a < and end with a >
- This is the opening tag

- Opening tags always starts with a tag name, it's p in this case which stands for paragraph

- All HTML tags start with an 'lt' angle bracket and end with a 'gt' bracket

- Denotes the beginning and end of a tag

HTML Tag Anatomy

<p id="unique" class="group">Some Content</p>

This is the closing tag.

  • Most (but not all) HTML tags have a closing tag
  • Starts with forward slash ( / ) followed by the tag name ( p ) in this case
- (Read slide)

- So if the tag starts as a p tag, it ends with a p tag.

- It's important to remember the forward slash in the closing tag, if you forget it - you're starting a new tag without closing the first one.

- ALWAYS remember forward slash when closing a tag

HTML Tag Anatomy

<p id="unique" class="group">Some Content</p>

This is the content of the tag.

  • The content sits between the opening and closing tags
  • This is the content that appears on your page
- (Read slide)

- So when browser looks at your markup, the tags give it context.

- In this case, it knows it's going to render a paragraph.

- It then looks at the content between your opening and closing tag and renders it to the page.

- Remember how we mentioned MOST tags have opening and closing tags? Since we know that the browser will only render the content between the opening and closing tags, that probably means the other type of tags that don't have a closing tag is not going to be rendering text to the screen.

HTML Tag Anatomy

<p id="unique" class="group">Some Content</p>

These are the attributes.

  • Attributes provide further description of the content or purpose of the tag
  • We can use these attributes to stype our page
  • Attributes are always structured as key="value"
- Inside of the opening tag and only inside of the opening tag is where you'll find attributes.

- Attributes always take this format: key = "value" (in quotes)

- Actually, in HTML5 - the quotes are optional but you'll see that almost every developer will still leave them in because it's better style and easier to read

- In fact, I've never seen anyone leave the quotes off - Rocco, Ellen?

- So technically you can, but don't. :P

HTML Tag Anatomy

<p id="unique" class="group">Some Content</p>

This is the ID attribute.

  • Uniquely identifies an HTML element in a document
  • You can add this attribute to any tag
  • Useful for styling and adding interaction
- You can add this attribute to any HTML element in your document

- The ID attribute's job is to uniquely identify a particular tag in the document

- That means in any given document, you can only have one p tag with the id of unique. You can't have multiple "unique" id tags.

- Have to name the id something else

- We'll be working more with id's when we get to the JS portion of this course: Example scenario - when you click on this specific button, you want a specific action to occur. ID's allow you to uniquely indentify that specific button because you may have multiple buttons on a page.

- You can also use it for styling purposes but it's generally not advised. We'll get into why next week but it has to do with CSS specificity

- To re-iterate, no two tags can have the same id in a given document. You use it to uniquely identify an element.

HTML Tag Anatomy

<p id="unique" class="group">Some Content</p>

This is the class attribute.

  • Enables you to group similar things together, like how cars and trucks are both automobiles
  • Useful for styling and adding interaction to many elements at once
- Classes are similar to ID's in that they're useful for styling and interaction purposes but that's where the similarities end

- ID's allow you to identify and select specific elements where as classes allow you to identify groups of elements

HTML Tag Anatomy

<img src="some-picture.jpg" />

Some tags shouldn't have closing tags. They're self-closing.

  • This image tag represents an image in our document
  • It has a src attribute that points to an image
  • Since the image is the content for this tag, we don't need a closing </img> tag - it's not allowed!
  • The / at the end is optional
- This is an example of a self-closing tag

- This image tag represnts an image in our document

- You'll notice there's a new attribute we haven't seen before, it's the src attribute. This src attribute points to an image.

- The key thing here is that since the image IS the content for this tag, we don't need a closing tag because there's no content to put between an opening and closing tag.

- The content is included through the src attribute.

- The img tag isn't the only self-closing tag. We'll come across more as we continue through the course

- And the / is optional. I like Including it, it's more clear that the tag has ended. It's up to you :)

HTML tags can be nested inside one another.

  • HTML is represented as a tree
  • You can put tags inside other tags as their content
  • The outer tag is the parent tag and the inner tag(s) are the children tags
- HTML has a hierarchical structure

- All the tags have a parent child relationship. Here you can see that the h1 and p tags are nested inside the article tag. The article is the PARENT. The p & h1 are the children.

- It's very much like a tree, we start with the root of the docuement, it's the the highest element up top and you branch out afterwards

- Many elements accept other HTML elements as their content, as you can see in article's case

- And you can nest as deeply as you want.
